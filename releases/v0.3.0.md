# v0.3.0 (beta)

## Highlights

This release expands `jedi-academy-pterodactyl` from a single TaystJK image into a
**multi-runtime image family**, and refactors the in-image runtime layout so all
runtimes share the same `/opt/jka` structure and a versioned `runtime.json`
manifest.

New selectable runtimes (each shipped as its own Docker image and Pterodactyl egg):

- **OpenJK modern64** — stable, recommended modern 64-bit OpenJK dedicated server
- **OpenJK legacy32** — 32-bit OpenJK build for legacy mod / `.so` compatibility
- **vanilla legacy32** — Raven 1.01 dedicated server (32-bit, user-supplied engine)
- **YBEProxy legacy32** — Yberion's JKA YBEProxy 32-bit dedicated server
- **TaystJK legacy32** *(experimental)* — 32-bit TaystJK build, gated behind
  `PUBLISH_TAYSTJK_LEGACY32`

The existing **TaystJK (modern64)** image remains the default and is unchanged
in behavior.

---

## Quick migration

Existing TaystJK users do **not** need to do anything — `latest`, the
`taystjk` tag, `master-<sha>`, `taystjk-master-<sha>`, and `vX.Y.Z` tags
still point at the same TaystJK modern64 image.

> **Note:** the TaystJK egg file was renamed from
> `egg/egg-taystjk-pterodactyl.json` to
> [`egg/egg-jka-taystjk-modern64-pterodactyl.json`](egg/egg-jka-taystjk-modern64-pterodactyl.json).
> If you re-import the egg, use the new path. Already-imported eggs and
> existing servers continue to work without changes.

To try one of the new runtimes:

1. In Pterodactyl go to **Admin → Nests → your nest → Import Egg** and import
   the egg JSON for the runtime you want from the [`egg/`](egg/) directory
   (e.g. `egg-openjk-modern64-pterodactyl.json`,
   `egg-openjk-legacy32-pterodactyl.json`, `egg-vanilla-legacy32-pterodactyl.json`,
   `egg-ybeproxy-legacy32-pterodactyl.json`).
2. Create a new server using that egg, or open an existing server in
   **Admin → Servers → \<server\> → Startup** and switch the Docker image to the
   matching tag, e.g.:
   - `ghcr.io/akiondev/jedi-academy-pterodactyl:openjk-modern64`
   - `ghcr.io/akiondev/jedi-academy-pterodactyl:openjk-legacy32`
   - `ghcr.io/akiondev/jedi-academy-pterodactyl:vanilla-legacy32`
   - `ghcr.io/akiondev/jedi-academy-pterodactyl:ybeproxy-legacy32`
3. Restart the server. Game data and config volumes are unchanged.

See [`docs/image-strategy.md`](../docs/image-strategy.md) for the full tag
table and [`docs/legacy32-operator-sheet.md`](../docs/legacy32-operator-sheet.md)
for legacy32-specific operator guidance.

### Repository layout

- Promoted **TaystJK modern64** to its own family directory for structural
  consistency with the other runtimes:
  - `docker/Dockerfile` → `docker/taystjk-modern64/Dockerfile`
  - `image/runtime.json` → `image/taystjk-modern64/runtime.json`
  - `egg/egg-taystjk-pterodactyl.json` → `egg/egg-jka-taystjk-modern64-pterodactyl.json`
- CI matrix target renamed from `taystjk` to `taystjk-modern64`; cache scope
  renamed from `taystjk-image` to `taystjk-modern64-image`. The published
  image tags (`latest`, `taystjk`, `master-<sha>`, `taystjk-master-<sha>`,
  and `vX.Y.Z` on tags) are unchanged for backward compatibility.
